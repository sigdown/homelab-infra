# Homelab Infrastracture (IaC)

Проект домашней гибридной лаборатории, основанный на концепции инфраструктура как кода (Infrastructure as Code). Включает в себя локальный гипервизор Proxmox и облачный Edge-сервер (Beget).

## Архитектура
- Локальная машина: Proxmox VE на физическом железе (Ryzen 3 3200G, 16 GB DDR4).
- Удаленный хост: VPS в облаке Beget (1 vCPU, GB RAM).

Облако с публичным белым IP помогает:
1. Решить проблему CGNAT, ограничивающую доступность локальной машины в интернете. 
2. Решить проблему безопасности соединения (открытие 443 и 80 портов на домашнем роутере считается нежелательной практикой).

## Технологеский стек
- `Terraform` - провижининг железа (Proxmox VE, Beget, генерация inventory для Ansible).
- `Ansible` - конфигурация (Base hardening, UFW, SSH будущий K3s).
- `Proxmox VE` - локальный гипервизор KVM/QEMU.

## Quick Start
1. Склонируйте репозиторий себе на машину.
2. Перейдите в папку `terraform` и создайте там файл с секретами:
```bash
cd terraform && vi terraform.tfvars
```
Структура файла с секретами должна выглядеть следующим образом:
```bash
pm_api_token_id = ""
pm_api_token_secret = ""
beget_api_token_secret = ""
```
3. Инициализировать terraform и выполнить планирование:
```bash
terraform init
terraform plan
```
4. Убедившись, что всё корректно, выполнить:
```bash
terraform apply
```
5. Перейти в `ansible/` и проверить `inventory.yml`. 
```bash
cd ansible && cat inventory.yml
```
6. Убедившись, что всё корректно, выполнить:
```bash
ansible-playbook playbooks/site.yml
```
Более подробная инструкция по полной настройке будет предоставлена в документации.

## Known Issues
В процессе эксплуатации схемы возможны некоторые "нюансы". Подробнее об этом будет сказано в документации.
1. **Динамический IP**. Провайдер Beget, к сожалению, пока не предоставил возможность аренды нод со статическими IP-адресами через Terraform API. Поэтому на данном этапе имеет смысл стараться не уничтожать лишний раз edge, потому что это вынудит вас менять адрес в доменных записях DNS. Возможные решения: использовать Cloudflare для автоматизации обновления записей.
2. **Cloud-init**. Можно было бы сократить часть кода на Ansible, но к сожалению, Beget Terraform API пока не дает возмоность использовать cloud-init для настройки серверов при их инициализации. Решение: сначала создаются ноды с пользователем root по симметричной схеме и потом Ansible создает sudo-пользователя отдельно на каждой ноде.
3. **QEMU Guest Agent**. По умолчанию в стандартном образе Ubuntu нет пакета qemu-guest-agent. Если в Terraform включить agent { enabled = true }, Proxmox не сможет получить IP-адрес ВМ. Решение: использование статических IP через Cloud-Init.
4. **Токены API Proxmox и RBAC (ошибка VM.Monitor)**: Старый Terraform-провайдер telmate/proxmox не поддерживает новые системы прав Proxmox 8.2+. Решение: использование современного провайдера bpg/proxmox.
5. **Invalid hostname**: Linux строго соблюдает RFC 952 и не разрешает использовать нижнее подчеркивание (_) в именах хостов. Используйте дефисы (-).
