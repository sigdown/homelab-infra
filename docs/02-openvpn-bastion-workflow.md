# Настройка OpenVPN для админского доступа

## Введение

Наш будущий кластер будет находится в локальной сети (по умолчанию `192.168.1.0/24`), доступ в которую обеспечивается через роутер с белым IP-адресом (отсутствует CGNAT). 

Но открывать публичный доступ до Proxmox API для Terraform и SSH для Ansible считается плохой практикой, так как интернет наполнен сканнерами, которые массово сканируют порты (особенно `22` порт, на котором работает админский доступ по SSH).

Гораздо лучше было бы создать приватную виртуальную сеть, чтобы из любой точки мира только администратор мог иметь доступ до локальной сети с кластером.

## Подготовка к настройке

В некоторых сетях UDP может работать нестабильно из-за ограничений провайдеров или особенностей маршрутизации. Поэтому для административного VPN удобнее использовать TCP, который OpenVPN поддерживает без проблем.

Прежде чем начать администрирование инфраструктуры, надо будет пробросить порт на роутере, поднять OpenVPN-сервер, создать пользователя и связать рабочую станцию с этим сервером (все эти действия выполняются в локальной домашней сети). 

## Проверка IP-адреса

Чтобы проверить является ли наш IP белым, выполните следующую команду:

```sh
curl ipinfo.io
```

Придёт ответ следующего вида:

```json
{
  "ip": "ip",
  "hostname": "hostname",
  "city": "city",
  "region": "region",
  ...
}
```
Сравните полученный адрес с адресом WAN в панели своего роутера. Если адреса совпадают - значит у вас белый адрес и можно продолжать настройку. Если адреса различаются, скорее всего, используется CGNAT, и входящие подключения из Интернета работать не будут.

## Настройка роутера
Зайдите в панель вашего роутера (по умолчанию `192.168.1.1` в строке браузера). Необходимо будет добавить правило на проброс порта:

```
protocol: TCP
source port: 1194
destination port: 1194
device ip: будущий локальный IP машины с OpenVPN-сервером (например, 192.168.1.99).
```
Роутер будет принимать запросы на `1194` порт (стандартный порт OpenVPN) и прокидывать их внутрь на сокет `openvpn_server_ip:1194` в локальную сеть. 

## Создание хоста для сервера
Здесь нам необходимо будет выполнить ряд действий. Самое первое из них - необходимо создать хост для OpenVPN-сервера. Используем LXC-контейнеры, которые из коробки присутствуют в Proxmox. Контейнер можно создать прямо из админской панели Proxmox (введите в адресную строку `proxmox_host_ip:proxmox_admin_port`).

Мы попали в админскую панель Proxmox. Получим шаблон для контейнера.

1. Получите образ. 

Разверните вкладку `pve`, перейдите в `local-pve` и дальше в `CT Templates`. Сверху выберите вкладку `Templates`. Скачайте желаемый образ (в примере будет использован `Debian 12`).

2. Создайте контейнер. 
- Справа сверху выберите `Create CT`. В `General `указывайте параметры: по вашему желанию. Главное чтобы `Node` был `pve` и `CT ID` был свободен. `Resource pool` оставляйте пустым. Контейнер должен быть непривилегированным и поддерживать `nesting` (вложенные контейнеры).
- В `Template` выберите полученный шаблон.
- В `Disks` выберите `local-lvm` и размер диска.
- В `CPU` количество виртуальных ядер процессора.
- В `Memory` количество памяти и размер `swap`.
- В `Network` всё по умолчанию, кроме `IPv4/CIDR` и `Gateway`. В `IPv4/CIDR` указывайте строго то, что прописали при форвардинге портов на роутере и добавьте CIDR (в нашем случае это `192.168.1.99/24`), а в `Gateway` локальный IP вышего шлюза (по умолчанию это `192.168.1.1`). 

Чтобы точно узнать IP шлюза посмотрите:

```bash
ip route | grep default
```
Ответ будет следующий по форме, но не обязательно аналогичный по содержанию.

```bash
default via 192.168.1.1 dev wlp2s0 proto dhcp src 192.168.1.108 metric 600 
```

Первый адрес и есть я IP шлюза.

3. Дальше просто оставьте всё по умолчанию. На данном этапе эти параметры не так важны. 

У нас есть готовы контейнер.

## Монтирование интерфейса tun

В Unix-подобных системах любой сетевой интерфейс является файлом. Для того что OpenVPN-сервер смог работать, необходимо примонтировать этот интерфейс прямо в контейнер. Также надо дать разрешение на создание символьных устройств с мажорным номером `10` и минорным номером `200`. Необходимо зайти в консоль самого Proxmox.

Откройте файл `/etc/pve/lxc/container_id.conf` в редакторе:

```bash
nano /etc/pve/lxc/container_id.conf
```

И добавьте следующие строки:

```bash
lxc.cgroup2.devices.allow: c 10:200 rwm
lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file
```

Перезапустите контейнер:

```bash
pct reboot <container_id>
```
## Подготовка окружения

Подключитесь к контейнеру через веб-консоль Proxmox либо командой `pct enter <container_id>` из терминала. Нам необходимо будет подготовить окружение для работы `OpenVPN-сервера`. 

1. Обновите систему, установите нужные утилиты и сделайте форвардинг IP.
```bash
apt update && apt upgrade -y

apt install -y curl wget git htop net-tools dnsutils ca-certificates

echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf && sysctl -p
```
2. Установите Docker согласно официальной документации по установке: https://docs.docker.com/engine/install/debian/
3. Проверьте работу Docker:
```bash
docker run --rm hello-world
```
## Настройка OpenVPN-сервера
После подготовки окружения надо установить сервер OpenVPN. Используем Docker Compose для удобной настройки контейнера с сервером.

1. Создадим директорию для `OpenVPN`:
```bash
cd /opt && mkdir -p openvpn && cd openvpn/
```
2. Создадим `docker-compose.yml`. Обратите внимение, что используется системный `network_host`.
```yml
services:
  openvpn:
    image: kylemanna/openvpn
    container_name: openvpn
    restart: unless-stopped

    network_mode: host

    cap_add:
      - NET_ADMIN

    devices:
      - /dev/net/tun:/dev/net/tun

    volumes:
      - ./openvpn-data:/etc/openvpn
```
3. На этом этапе сгенерировать сертификаты и инициализировать PKI. Введите в терминал следующие команды.

```bash
# Генерация пароля для сертификата
openssl rand -base64 12

# Генерация конфига для OpenVPN
docker run --rm \
-v $PWD/openvpn-data:/etc/openvpn \
kylemanna/openvpn \
ovpn_genconfig \
-u tcp://your_router_ip \
-s 10.8.0.0/24

# Инициализация PKI
docker run --rm -it \
-v $PWD/openvpn-data:/etc/openvpn \
kylemanna/openvpn \
ovpn_initpki

# Запуск контейнера
docker compose up -d
```
4. Добавьте пользователя (клиентский сертификат + ключ).
```bash
docker run --rm -it \
-v $PWD/openvpn-data:/etc/openvpn \
kylemanna/openvpn \
easyrsa build-client-full laptop nopass
```
5. Сгенерируйте `.ovpn` файл для пользователя.
```bash
docker run --rm \
-v $PWD/openvpn-data:/etc/openvpn \
kylemanna/openvpn \
ovpn_getclient laptop > laptop.ovpn
```
6. Добавьте правило `iptables` на маскардинг и форвардинг подсети. Это обязательный шаг. Без этих правил пакеты из VPN-подсети не будут маршрутизироваться и клиенты не смогут получить доступ к локальной сети.

```bash
# Разрешить форвардинг подсети VPN в iptables хоста
iptables -I FORWARD 1 -s 10.8.0.0/24 -j ACCEPT
iptables -I FORWARD 2 -d 10.8.0.0/24 -m state --state RELATED,ESTABLISHED -j ACCEPT

# Включить NAT
iptables -t nat -I POSTROUTING 1 -s 10.8.0.0/24 -o eth0 -j MASQUERADE

# Сохранить правила
iptables-save > /etc/iptables/rules.v4
```
7. Получите файл с хоста.
```bash
# На хосте Proxmox
pct pull <container_id> /opt/openvpn/laptop.ovpn ~/laptop.ovpn

# На рабочей машине
scp root@proxmox_host_ip:/root/laptop.ovpn ~/Downloads
```
## Настройка OpenVPN-клиента
Далее нам необходимо установить клиент для работы с OpenVPN. Самый простой способ - поставить `openvpn` на рабочую станцию и выполнить соединение, можно в отдельной `tmux` сессии. 

Для установки клиента на Windows и MacOS ознакомьтесь с официальной документацией.

Установка на разные дистрибьютивы Linux:

```bash
# На Arch
sudo pacman -S openvpn tmux --needed

# На Debian
sudo apt install openvpn tmux

# На Fedora
sudo dnf install openvpn tmux
```
1. Создайте новую tmux-сессию и зайдите в неё:
```bash
tmux new -t openvpn

tmux attach -t openvpn
```
2. Создайте новое соединение с OpenVPN сервером:
```bash
sudo openvpn --config ~/Downloads/laptop.ovpn --client
```
3. Проверьте маршруты через `ip route`.
